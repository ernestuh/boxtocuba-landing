Part 3 — CMS structure (Supabase)
One JSON key controls everything — on/off, text, affected provinces
settings › key: LANDING_NOTICE
{
  "enabled": true,

  // ⚠️ Capital only — no direct flights
  "capital_only": [
    "cienfuegos", "sancti-spiritus",
    "ciego-de-avila", "camaguey",
    "las-tunas", "holguin",
    "granma", "santiago-de-cuba"
  ],

  // 📍 Pickup in Havana
  "pickup_havana": [
    "artemisa", "mayabeque",
    "pinar-del-rio", "isla-de-la-juventud"
  ],

  // 🚫 Pickup in Santiago
  "pickup_santiago": [
    "guantanamo"
  ]
}
✅ To activate
Set "enabled": true → redeploy → all banners + table indicators appear automatically.
🔕 To remove notices
Set "enabled": false → redeploy → site returns to normal. Zero code changes needed.
✏️ Partial recovery
When flights resume for a province, remove its slug from the list and redeploy. Other provinces keep their notice unaffected.
